Findings

▶ Finding 1: Login Page — All Elements Undersized + Auto-Zoom Critical

Every interactive element on the login page falls below the iOS 44px minimum touch target. Input font sizes of 14px trigger Safari's auto-zoom on focus, degrading the mobile experience.

Login Page
  • “Create account” link — 17px height
  • Email input — 38px height / 14px font
  • Password input — 38px height / 14px font
  • Sign In button — 36px height / 14px font
  • Google OAuth button — 38px height / 14px font
  • GitHub OAuth button — 38px height / 14px font
Root Cause Default shadcn/ui sizing without mobile overrides. All components inherit desktop-sized defaults that do not meet Apple HIG touch target minimums.
Recommended Fix Set font-size: 16px on all inputs (prevents Safari auto-zoom). Apply min-height: 44px to all interactive elements: inputs, buttons, and links.
▶ Finding 2: Dashboard Activity Filter Tabs — 24px Height High

The 6 activity filter tabs (All, Check-ins, Notes, Milestones, Actions, Requests) render at only 24px — nearly half the 44px Apple HIG minimum. The “Show more” button below is also undersized at 34px.

Dashboard Top
Activity Tabs (24px)
  • “All” filter tab — 24px height (20px below minimum)
  • “Check-ins” filter tab — 24px height
  • “Notes” filter tab — 24px height
  • “Milestones” filter tab — 24px height
  • “Actions” filter tab — 24px height
  • “Requests” filter tab — 24px height
  • “Show more” button — 34px height
Recommended Fix Add min-h-[44px] to all filter tab buttons and the “Show more” button component.
▶ Finding 3: Check-in Edit Button — 40px Height Low

The “Edit” button on the check-in page measures 40px — only 4px below the 44px minimum. This is a minor deviation that may still cause occasional mis-taps.

Check-in Page
  • “Edit” button — 40px height (4px below minimum)
Recommended Fix Add min-h-[44px] to the Edit button.
