Critical Finding: Horizontal Overflow on Mobile

The most significant issue discovered is horizontal overflow on 5 of 8 pages at the iPhone 15 Pro viewport (393px). Content extends far beyond the viewport, enabling accidental horizontal scrolling that reveals broken whitespace layout. This affects the header, main content, and tab bar.

Root Cause

High — Layout

App Shell flex container not constraining width on mobile

In src/app/(app)/app-shell.tsx:20, the main content wrapper flex-1 lg:ml-64 flex flex-col does not constrain its width to the viewport. Child elements (header, main, PageContainer with max-w-6xl) expand the flex item beyond 393px. The parent flex min-h-screen container has no overflow-x: hidden.

Contributing elements per page:

  • Dashboard (746px): Decorative absolute -right-6 circle in prep banner extends to 1155px; card grids and widgets expand unconstrained
  • Settings (553px): Horizontal tab strip and form content expand layout to 946px
  • Reminders (335px): Chat-style UI and max-w-6xl container expands to 728px
  • Growth (86px): Filter button row flex items-center gap-2 extends slightly
  • Love Languages (78px): max-w-6xl PageContainer pushes minor overflow

Fix: Add overflow-x: hidden to the app shell outer container or the flex-1 wrapper. Also consider adding max-w-full or w-0 min-w-0 to prevent flex children from exceeding viewport bounds.
